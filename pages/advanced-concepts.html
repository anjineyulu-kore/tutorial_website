1. Object-Oriented Programming (OOP) in Testing

OOP is fundamental to writing maintainable, scalable test automation code. Let's explore the four pillars of OOP with practical testing examples.

ğŸ“ The Four Pillars of OOP

ğŸ¯ Encapsulation

Bundle data and methods together, hiding internal implementation details.

Example: A login page class encapsulates username field, password field, and login method.

ğŸ”„ Inheritance

Child classes inherit properties and methods from parent classes.

Example: LoginPage inherits common functions from BasePage class.

ğŸ­ Polymorphism

Objects can be used in many forms; same method, different behaviors.

Example: click() method works on button, link, and checkbox differently.

ğŸ—ï¸ Abstraction

Show only essential features, hide complex implementation details.

Example: WebDriver.find_element() abstracts browser finding logic.

Purpose: Encapsulate page elements and actions in separate classes

4. Page Object Model - Deep Dive

The Page Object Model is the most popular design pattern for web automation. It separates test logic from page element locators.

ğŸ¯ POM Structure Best Practices

1
Define Page Elements

Use tuples with By locator to define all page elements at class level.

class LoginPage(BasePage): USERNAME_FIELD = (By.ID, "username") PASSWORD_FIELD = (By.NAME, "password") LOGIN_BUTTON = (By.XPATH, "//button[@type='submit']") ERROR_MESSAGE = (By.CLASS_NAME, "error")
2
Create Action Methods

Create methods for user interactions without using locators in tests.

def login(self, username, password): self.find_element(self.USERNAME_FIELD).send_keys(username) self.find_element(self.PASSWORD_FIELD).send_keys(password) self.click(self.LOGIN_BUTTON) def get_error_message(self): return self.find_element(self.ERROR_MESSAGE).text
3
Use in Tests

Tests should be simple and readable, using only page methods.

def test_valid_login(driver): login_page = LoginPage(driver) home_page = login_page.login("user@test.com", "password123") assert home_page.is_dashboard_visible() def test_invalid_login(driver): login_page = LoginPage(driver) login_page.login("user@test.com", "wrongpass") assert "Invalid credentials" in login_page.get_error_message()

ğŸ“‹ POM Best Practices Checklist

âœ… Do's

  • Keep locators at class level
  • One method per user action
  • Use meaningful method names
  • Return page objects from methods
  • Inherit from BasePage
  • Use explicit waits

âŒ Don'ts

  • Don't use locators in tests
  • Don't create methods for assertions
  • Don't hardcode wait times
  • Don't use implicit waits
  • Don't make methods do too much
  • Don't duplicate locators

import functools import time def retry(max_attempts=3, delay=1): """Decorator to retry failed test functions""" def decorator(func): @functools.wraps(func) def wrapper(*args, **kwargs): for attempt in range(max_attempts): try: return func(*args, **kwargs) except Exception as e: if attempt == max_attempts - 1: raise print(f"Attempt {attempt + 1} failed, retrying in {delay}s...") time.sleep(delay) return wrapper return decorator # Usage @retry(max_attempts=3, delay=2) def test_flaky_feature(): # Test code that might fail occasionally assert some_condition()

8. Debugging & Troubleshooting

ğŸ” Common Issues & Solutions

âŒ NoSuchElementException

Cause: Element not found on page

Solution:

  • Verify locator is correct
  • Add explicit waits
  • Check if element is in iframe
  • Verify element visibility

âŒ StaleElementReferenceException

Cause: DOM changed after element reference

Solution:

  • Re-find element before interaction
  • Use dynamic locators
  • Catch and retry
  • Avoid storing element references

âŒ TimeoutException

Cause: Element not found within timeout

Solution:

  • Increase timeout value
  • Check network/server issues
  • Verify element will appear
  • Check browser capabilities

âŒ ElementNotClickableException

Cause: Element not clickable or not visible

Solution:

  • Scroll to element
  • Use ActionChains to click
  • Wait for element to be clickable
  • Check for overlaying elements

âŒ Flaky Tests

Cause: Tests fail intermittently

Solution:

  • Use explicit waits properly
  • Avoid hard-coded delays
  • Handle async operations
  • Implement retry logic

âŒ WebDriver Session Issues

Cause: Driver crashed or lost connection

Solution:

  • Implement proper cleanup
  • Use try-finally blocks
  • Close all windows
  • Clear browser cache
