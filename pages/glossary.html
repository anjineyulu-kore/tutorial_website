üéØ General Testing Terms

Assertion

Testing

A statement that verifies whether a condition is true. If false, the test fails. Assertions are the core of validation in automated tests.

assert actual_value == expected_value assert user.is_logged_in() == True assert response.status_code in [200, 201]
See also: Test Case Validation

Test Case

Testing

A set of conditions or steps used to verify a specific functionality. Each test case has inputs, steps, and expected outcomes.

def test_user_login(): # Setup # Execute # Assert

Test Fixture

Testing

A setup of preconditions before running a test. Includes database initialization, test data creation, and resource allocation.

@pytest.fixture def user_data(): return {"name": "John", "email": "john@test.com"}

Mock / Mocking

Testing

Creating a fake object that simulates real behavior. Used to isolate units of code and test without external dependencies.

Stub

Testing

A minimal implementation of a function or method. Returns hardcoded values without actual logic.

Test Coverage

Testing

Percentage of code executed by tests. High coverage (>80%) indicates more functionality is being tested.

Flaky Test

Testing

A test that produces inconsistent results - sometimes passing, sometimes failing under same conditions. Usually caused by timing issues or external dependencies.

Test Suite

Testing

A collection of multiple test cases grouped together. Can be organized by feature, module, or functionality.

üêç Python Concepts

Virtual Environment

Python

An isolated Python environment where you can install packages without affecting the system Python. Essential for project dependency management.

python -m venv env source env/bin/activate # Linux/Mac env\Scripts\activate # Windows

Decorator

Python

A function that modifies another function or class. Used to add functionality without changing the original code.

@pytest.fixture @retry(max_attempts=3) def test_function(): pass

List Comprehension

Python

Concise way to create lists using a single line of code. More efficient than using loops.

# Traditional results = [] for test in tests: results.append(test.run()) # List comprehension results = [test.run() for test in tests]

Context Manager

Python

Manages setup and teardown of resources using 'with' statement. Ensures resources are properly released.

with open('file.txt') as f: data = f.read() # File auto-closes

Lambda Function

Python

Anonymous function defined in one line. Useful for simple operations and callbacks.

# Regular function def double(x): return x * 2 # Lambda equivalent double = lambda x: x * 2

Generator

Python

A function that returns values one at a time using 'yield'. Memory efficient for large datasets.

def test_data_generator(): for i in range(100): yield {"id": i, "value": i*2}

üåê Selenium & Web Automation

Locator

Selenium

A strategy to find elements on a web page. Types include ID, CSS Selector, XPath, Name, Class, etc.

# By ID (By.ID, "username") # By CSS Selector (By.CSS_SELECTOR, ".login-button") # By XPath (By.XPATH, "//button[@type='submit']")

XPath

Selenium

XML Path Language used to navigate and select elements in a DOM. Most powerful but slower locator strategy.

# Absolute XPath /html/body/div/form/input # Relative XPath (Recommended) //input[@id="username"] //button[contains(text(), 'Login')]

WebDriver

Selenium

The core interface that communicates with the browser. Provides methods to navigate, find elements, and perform actions.

driver = webdriver.Chrome() driver.get("https://example.com") driver.find_element(By.ID, "username")

WebDriverWait

Selenium

Explicitly waits for an element to appear or condition to be true before proceeding. Prevents timing issues.

wait = WebDriverWait(driver, 10) element = wait.until(EC.presence_of_element_located((By.ID, "button")))

Page Object Model (POM)

Selenium

Design pattern that creates a class for each page/component. Encapsulates locators and page interactions.

Stale Element Reference

Selenium

Error when an element reference is no longer attached to the DOM. Occurs when page reloads or DOM changes.

ActionChains

Selenium

Class for simulating complex user interactions like mouse movements, drag-drop, and multi-key presses.

actions = ActionChains(driver) actions.move_to_element(element) actions.click() actions.perform()

üì° API Testing

REST API

API

Representational State Transfer API. Uses standard HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources.

HTTP Status Code

API

Three-digit code indicating the result of an HTTP request. 2xx=Success, 3xx=Redirect, 4xx=Client Error, 5xx=Server Error.

JSON (JavaScript Object Notation)

API

Lightweight data format for transmitting structured data. Human-readable and widely used in APIs.

{ "name": "John", "age": 30, "active": true }

Request Header

API

Metadata sent with an HTTP request. Contains information like Content-Type, Authorization, etc.

headers = { "Content-Type": "application/json", "Authorization": "Bearer token123" }

Request Body

API

Data sent with HTTP request. Used in POST and PUT requests to send data to the server.

Response

API

Data returned by the server after processing a request. Contains status code, headers, and body.

OAuth

API

Open standard for authentication and authorization. Allows secure access to resources without exposing passwords.

ü§ñ Robot Framework

Keyword

Robot

A reusable action in Robot Framework. Keywords are either built-in, from libraries, or user-defined.

Test Case

Robot

A sequence of keywords that test a specific feature. Structured in a readable table format.

Test Suite

Robot

A directory or file containing test cases. Can be nested to create hierarchical test structure.

Variable

Robot

A named container for data. Types include scalars, lists, and dictionaries. Prefixed with $ for scalars.

${USERNAME} admin @{BROWSERS} Chrome Firefox &{USER} name=John age=30

Library

Robot

A collection of keywords. Built-in libraries (BuiltIn, Collections) or external (SeleniumLibrary, RequestsLibrary).

üé≠ Playwright

Playwright Browser

Playwright

Browser automation framework supporting Chromium, Firefox, and WebKit. Provides unified API across all browsers.

Page Object

Playwright

Represents a webpage. Provides methods to interact with elements, navigate, and retrieve page information.

Locator

Playwright

A query language to find elements. Supports CSS, XPath, and Playwright-specific selectors.

Network Interception

Playwright

Ability to monitor, mock, and modify network requests. Used for testing error scenarios and API responses.

üîÑ Jenkins & CI/CD

Jenkins Pipeline

Jenkins

Defines build, test, and deployment process as code. Two types: Declarative (simple) and Scripted (flexible).

Declarative Pipeline

Jenkins

Simplified pipeline syntax with predefined structure. Easier to read and maintain.

Scripted Pipeline

Jenkins

Groovy-based pipeline syntax. More powerful and flexible but requires programming knowledge.

Build Stage

Jenkins

A distinct phase in the pipeline (Build, Test, Deploy). Each stage has specific steps and responsibilities.

Job

Jenkins

A unit of work in Jenkins. Can be Freestyle (UI-configured) or Pipeline (code-based).

Webhook

Jenkins

Trigger that activates Jenkins job when event occurs (e.g., GitHub push). Automates CI/CD workflow.

üî§ Common Abbreviations

API

Application Programming Interface - Interface for software to communicate with each other

QA

Quality Assurance - Process of ensuring software meets quality standards

CI/CD

Continuous Integration/Continuous Deployment - Automated testing and deployment pipeline

E2E

End-to-End Testing - Testing complete user workflows from start to finish

TDD

Test Driven Development - Write tests before implementing functionality

BDD

Behavior Driven Development - Write tests in human-readable format (Gherkin)

UI

User Interface - Visual elements users interact with in an application

DOM

Document Object Model - Tree structure representing HTML elements

CSS

Cascading Style Sheets - Used to locate elements; also for styling web pages

JSON

JavaScript Object Notation - Data format for APIs and configuration

HTTP

HyperText Transfer Protocol - Protocol for transferring data over web

REST

Representational State Transfer - Architectural style for web APIs

OAuth

Open Authorization - Protocol for secure authentication and authorization

SSH

Secure Shell - Protocol for secure remote access to servers

VM

Virtual Machine - Virtualized computer system running on physical hardware

IDE

Integrated Development Environment - Software for writing and debugging code

YAML

YAML Ain't Markup Language - Human-readable data format for configuration

XML

eXtensible Markup Language - Markup language for storing structured data

HTTP 200

OK - Request succeeded, response contains data

HTTP 404

Not Found - Requested resource doesn't exist on server

HTTP 500

Internal Server Error - Server encountered unexpected error processing request

POM

Page Object Model - Design pattern for organizing test code by pages
