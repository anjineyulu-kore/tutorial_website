1. Introduction to Python

Python is one of the most popular programming languages for test automation due to its simplicity, readability, and powerful testing frameworks. It's used by QA engineers, DevOps professionals, and automation specialists worldwide.

Why Python for Test Automation?

  • Easy to Learn: Simple syntax that's easy to read and understand
  • Powerful Libraries: Rich ecosystem with Selenium, Requests, PyTest, Robot Framework
  • Cross-Platform: Runs on Windows, Mac, and Linux
  • Community Support: Large community with extensive documentation
  • Integration: Works seamlessly with CI/CD pipelines

2. Setup & Installation

Download and Install Python

  1. Visit python.org
  2. Download Python 3.8 or higher
  3. Run the installer and check "Add Python to PATH"
  4. Verify installation in terminal:
python --version pip --version

Install Essential Testing Libraries

# Install Pytest pip install pytest # Install Requests for API testing pip install requests # Install Selenium for web automation pip install selenium # Install Playwright pip install playwright # Install Robot Framework pip install robotframework # Install all in one command pip install pytest requests selenium playwright robotframework

3. Python Basics for Testing

Variables and Data Types

# Strings name = "John" message = 'Hello, World!' # Numbers age = 30 price = 19.99 # Lists fruits = ["apple", "banana", "orange"] numbers = [1, 2, 3, 4, 5] # Dictionaries user = {"name": "John", "age": 30, "email": "john@example.com"} # Tuples (immutable) coordinates = (10, 20) # Boolean is_active = True is_deleted = False

Conditional Statements

age = 25 if age >= 18: print("You are an adult") elif age >= 13: print("You are a teenager") else: print("You are a child") # Ternary operator status = "active" if age >= 18 else "inactive"

Loops

# For loop fruits = ["apple", "banana", "orange"] for fruit in fruits: print(fruit) # While loop counter = 0 while counter < 5: print(f"Count: {counter}") counter += 1 # Loop with range for i in range(1, 6): # 1 to 5 print(f"Number: {i}")

Functions

# Simple function def greet(name): return f"Hello, {name}!" result = greet("Alice") print(result) # Output: Hello, Alice! # Function with default parameters def add(a, b=5): return a + b print(add(10)) # Output: 15 print(add(10, 20)) # Output: 30 # Function with multiple returns def get_user_info(): return "John", 30, "john@example.com" name, age, email = get_user_info()

String Operations

text = "Hello, Python!" # String methods print(text.lower()) # hello, python! print(text.upper()) # HELLO, PYTHON! print(text.replace("Python", "World")) # Hello, World! # String formatting name = "Alice" age = 25 print(f"Name: {name}, Age: {age}") # f-string (modern) print("Name: {}, Age: {}".format(name, age)) # format method # String slicing substring = text[0:5] # "Hello" first_char = text[0] # "H" last_char = text[-1] # "!"

4. Object-Oriented Programming

Classes and Objects

class TestCase: def __init__(self, name, status): self.name = name self.status = status self.result = None def run_test(self): print(f"Running test: {self.name}") self.result = "PASSED" return self.result def get_status(self): return self.status # Create instance test = TestCase("Login Test", "Active") print(test.run_test()) # Running test: Login Test # Access attributes print(test.name) # Login Test print(test.result) # PASSED

Inheritance

class BaseTest: def __init__(self, test_name): self.test_name = test_name def setup(self): print(f"Setting up {self.test_name}") def teardown(self): print(f"Tearing down {self.test_name}") class LoginTest(BaseTest): def __init__(self, test_name): super().__init__(test_name) def test_login(self): self.setup() print("Testing login functionality") self.teardown() test = LoginTest("User Login") test.test_login()

Collections Module

from collections import Counter, namedtuple # Counter for counting occurrences test_results = ["PASS", "PASS", "FAIL", "PASS", "SKIP"] result_count = Counter(test_results) print(result_count) # Counter({'PASS': 3, 'FAIL': 1, 'SKIP': 1}) # Named tuple for structured data TestResult = namedtuple("TestResult", ["name", "status", "duration"]) result = TestResult("Login Test", "PASSED", 2.5) print(result.name) # Login Test print(result.status) # PASSED

JSON Module

import json # Parse JSON string json_data = '{"name": "John", "age": 30, "active": true}' data = json.loads(json_data) print(data["name"]) # John # Convert Python object to JSON user_data = { "name": "Alice", "age": 25, "skills": ["Python", "Selenium", "Testing"] } json_string = json.dumps(user_data, indent=2) print(json_string) # Read JSON from file with open("data.json", "r") as file: data = json.load(file) # Write JSON to file with open("output.json", "w") as file: json.dump(user_data, file, indent=2)

6. Testing Frameworks

Pytest - Modern Testing Framework

import pytest def test_addition(): assert 2 + 2 == 4 assert 2 + 2 != 5 def test_string_operations(): text = "Hello" assert text.lower() == "hello" assert len(text) == 5 class TestCalculator: def setup_method(self): self.calc_result = 0 def test_add(self): self.calc_result = 2 + 2 assert self.calc_result == 4 def test_subtract(self): self.calc_result = 5 - 2 assert self.calc_result == 3 # Run with: pytest test_file.py -v

7. Best Practices

Code Organization

  • Use meaningful variable names: use `test_result` instead of `tr`
  • Follow PEP 8: Use 4 spaces for indentation, keep lines under 80 characters
  • Write docstrings: Document your functions and classes
  • Keep functions small: Each function should do one thing
  • Use type hints: Help with code clarity and IDE support

Example: Well-Structured Test

""" Module for user authentication tests """ import pytest import requests class TestUserAuthentication: \"\"\"Test suite for user authentication functionality\"\"\" BASE_URL = "https://api.example.com" def setup_method(self): \"\"\"Setup before each test\"\"\" self.session = requests.Session() def teardown_method(self): \"\"\"Cleanup after each test\"\"\" self.session.close() def test_valid_login(self): \"\"\"Test successful login with valid credentials\"\"\" credentials = { "username": "testuser", "password": "securepass123" } response = self.session.post( f"{self.BASE_URL}/auth/login", json=credentials ) assert response.status_code == 200 assert "token" in response.json() def test_invalid_password(self): \"\"\"Test login failure with invalid password\"\"\" credentials = { "username": "testuser", "password": "wrongpass" } response = self.session.post( f"{self.BASE_URL}/auth/login", json=credentials ) assert response.status_code == 401

Virtual Environments

Always use virtual environments to isolate project dependencies:

# Create virtual environment python -m venv venv # Activate on Windows venv\Scripts\activate # Activate on macOS/Linux source venv/bin/activate # Install dependencies pip install -r requirements.txt # Deactivate deactivate

Requirements File

Create a requirements.txt to manage dependencies:

pytest==7.4.3 selenium==4.15.2 requests==2.31.0 playwright==1.40.0 robotframework==6.1.1 python-dotenv==1.0.0

Error Handling

try: response = requests.get("https://api.example.com/data", timeout=5) response.raise_for_status() # Raise exception for bad status codes data = response.json() except requests.exceptions.Timeout: print("Request timed out") except requests.exceptions.ConnectionError: print("Connection failed") except ValueError: print("Invalid JSON response") except Exception as e: print(f"Unexpected error: {e}") else: print("Request successful") finally: print("Cleanup code here")
