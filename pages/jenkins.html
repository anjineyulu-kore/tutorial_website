1. Introduction to Jenkins CI/CD

Jenkins is an open-source automation server that helps automate parts of software development like building, testing, and deploying. It's widely used for implementing Continuous Integration (CI) and Continuous Deployment (CD) pipelines.

What is CI/CD?

  • Continuous Integration (CI): Automatically build and test code changes frequently
  • Continuous Deployment (CD): Automatically deploy tested code to production
  • Continuous Delivery: Prepare code for production deployment

Why Jenkins for Test Automation?

  • Free & Open Source: No licensing costs
  • Extensible: 1000+ plugins for various tools
  • Distributed: Run tests across multiple agents
  • Pipeline Support: Sophisticated workflow definitions
  • Wide Adoption: Industry standard in many companies
  • Integration: Works with Git, Docker, Kubernetes, and more

2. Installation & Setup

Install Jenkins on Windows

# Option 1: Using Installer # Download from: https://www.jenkins.io/download/ # Run the Windows installer # Jenkins will start on http://localhost:8080 # Option 2: Using Java directly # First install Java JDK 11 or higher # Download jenkins.war from jenkins.io java -jar jenkins.war

Install Jenkins on Linux/macOS

# Ubuntu/Debian wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo apt-key add - sudo sh -c 'echo deb https://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list' sudo apt-get update sudo apt-get install jenkins # Start Jenkins sudo systemctl start jenkins sudo systemctl enable jenkins # macOS (using Homebrew) brew install jenkins brew services start jenkins

Initial Setup

  1. Open browser: http://localhost:8080
  2. Get initial admin password from: /var/lib/jenkins/secrets/initialAdminPassword
  3. Install recommended plugins
  4. Create first admin user
  5. Configure Jenkins URL

Essential Plugins for Test Automation

# Install these plugins: - Git Plugin (for version control) - Pipeline (for declarative/scripted pipelines) - Python Plugin (for Python tests) - Robot Framework Plugin (for Robot tests) - JUnit Plugin (for test reporting) - Selenium Plugin - Email Plugin (for notifications) - Docker Plugin (for containerized tests) - Blue Ocean (for pipeline visualization)

3. Jenkins Basics

Jenkins Architecture

  • Master/Controller: Manages jobs and coordinates agents
  • Agents/Nodes: Execute jobs (jobs are distributed here)
  • Jobs: Work units that run builds
  • Builds: Individual executions of a job
  • Plugins: Extend Jenkins functionality

Job Lifecycle

1. Trigger ‚îú‚îÄ Manual trigger ‚îú‚îÄ SCM polling (GitHub webhook) ‚îú‚îÄ Build trigger (another job) ‚îî‚îÄ Scheduled trigger (cron) 2. Build ‚îú‚îÄ Checkout source code ‚îú‚îÄ Install dependencies ‚îú‚îÄ Build application ‚îî‚îÄ Run tests 3. Post-Build Actions ‚îú‚îÄ Archive artifacts ‚îú‚îÄ Generate reports ‚îú‚îÄ Send notifications ‚îî‚îÄ Deploy (if tests pass)

Job Configuration Basics

  • General: Job name, description, parameters
  • Source Code: Git, SVN, etc.
  • Build Triggers: When to run the job
  • Build Steps: Commands to execute
  • Post-Build: Actions after build completes

5. Jenkins Pipelines

Declarative Pipeline

pipeline { agent any stages { stage('Checkout') { steps { checkout scm } } stage('Install Dependencies') { steps { sh 'pip install -r requirements.txt' } } stage('Run Tests') { steps { sh 'pytest tests/ -v' } } stage('Generate Report') { steps { junit 'test-results.xml' } } } post { always { echo 'Cleaning up...' } success { echo 'Tests passed!' } failure { echo 'Tests failed!' } } }

Pipeline with Multiple Stages

pipeline { agent any options { timestamps() timeout(time: 1, unit: 'HOURS') buildDiscarder(logRotator(numToKeepStr: '10')) } environment { PYTHON_ENV = 'venv' TEST_REPORTS = 'reports/' } stages { stage('Setup') { steps { sh 'python -m venv ${PYTHON_ENV}' sh '. ${PYTHON_ENV}/bin/activate && pip install -r requirements.txt' } } stage('Code Quality') { steps { sh 'pylint src/' } } stage('Unit Tests') { steps { sh 'pytest tests/unit -v --junitxml=${TEST_REPORTS}unit-results.xml' } } stage('Integration Tests') { steps { sh 'pytest tests/integration -v --junitxml=${TEST_REPORTS}integration-results.xml' } } stage('Selenium Tests') { steps { sh 'pytest tests/selenium -v --junitxml=${TEST_REPORTS}selenium-results.xml' } } stage('API Tests') { steps { sh 'pytest tests/api -v --junitxml=${TEST_REPORTS}api-results.xml' } } } post { always { junit testResults: '${TEST_REPORTS}*.xml', allowEmptyResults: true publishHTML([ reportDir: 'htmlcov', reportFiles: 'index.html', reportName: 'Code Coverage' ]) } } }

6. Integrating Tests in Jenkins

Pytest Integration

#!/bin/bash # Install test dependencies pip install pytest pytest-cov pytest-xdist # Run tests with JUnit XML output pytest tests/ -v \ --junitxml=test-results.xml \ --cov=src \ --cov-report=html \ --cov-report=xml # Run tests in parallel pytest tests/ -n auto -v --junitxml=test-results.xml

Selenium Tests in Jenkins

#!/bin/bash # Install dependencies pip install selenium pytest # Run Selenium tests pytest tests/selenium/ -v \ --junitxml=selenium-results.xml \ --tb=short # Generate Allure report pip install allure-pytest pytest tests/selenium/ --alluredir=allure-results allure generate allure-results -o allure-report

Robot Framework in Jenkins

#!/bin/bash # Install Robot Framework pip install robotframework robotframework-seleniumlibrary # Run Robot tests robot --outputdir results/ \ --xunit results/xunit.xml \ --log NONE \ --report NONE \ tests/ # Or in Jenkins Pipeline: sh ''' robot --outputdir results/ \\ --xunit results/xunit.xml \\ tests/ '''

API Tests in Jenkins

#!/bin/bash # Run API tests with pytest pip install requests pytest pytest-json-report pytest tests/api/ -v \ --junitxml=api-results.xml \ --html=api-report.html \ --self-contained-html # With environment variables export API_URL=https://api.example.com export API_KEY=secret_key_123 pytest tests/api/ -v --junitxml=api-results.xml

7. Test Automation Integration

Complete Selenium Pipeline

pipeline { agent any parameters { choice(name: 'BROWSER', choices: ['Chrome', 'Firefox', 'Safari'], description: 'Browser to test') choice(name: 'ENVIRONMENT', choices: ['dev', 'staging', 'prod'], description: 'Target environment') } environment { CHROME_BIN = '/usr/bin/google-chrome' GECKODRIVER = '/usr/local/bin/geckodriver' } stages { stage('Checkout') { steps { checkout scm } } stage('Setup') { steps { sh ''' python -m venv venv . venv/bin/activate pip install -r requirements.txt ''' } } stage('Run Selenium Tests') { steps { sh ''' . venv/bin/activate pytest tests/selenium/ -v \\ --junitxml=selenium-results.xml \\ -m browser_${BROWSER} \\ -m env_${ENVIRONMENT} ''' } } stage('Generate HTML Report') { steps { sh ''' . venv/bin/activate pytest tests/selenium/ \\ --html=report.html \\ --self-contained-html ''' } } } post { always { junit 'selenium-results.xml' publishHTML([ reportDir: '.', reportFiles: 'report.html', reportName: 'Selenium Test Report' ]) archiveArtifacts artifacts: 'screenshots/**', allowEmptyArchive: true } failure { echo 'Tests failed - Check logs and artifacts' } } }

Robot Framework Pipeline

pipeline { agent any stages { stage('Install Dependencies') { steps { sh ''' pip install robotframework pip install robotframework-seleniumlibrary pip install robotframework-requests ''' } } stage('Run Robot Tests') { steps { sh ''' robot --outputdir results/ \\ --log NONE \\ --report NONE \\ --variable BROWSER:chrome \\ --variable ENV:dev \\ tests/ ''' } } } post { always { robot outputPath: 'results', logFileName: 'log.html', reportFileName: 'report.html', otherFiles: 'output.xml' } } }

9. Best Practices

Jenkins Configuration Best Practices

  • Use Declarative Pipelines: Easier to read and maintain
  • Store Secrets Securely: Use Jenkins credentials store
  • Version Control Pipeline: Keep Jenkinsfile in Git
  • Use Environment Variables: For configuration management
  • Implement Timeouts: Prevent hanging builds
  • Parallel Execution: Run independent tests in parallel
  • Clean Up: Archive artifacts and clean workspace
  • Notifications: Alert on success/failure

Test Automation Best Practices

  • Separate Test Suites: Unit, API, UI, Integration
  • Use Test Markers: Tag tests for selective execution
  • Parallel Testing: Run tests concurrently
  • Headless Browsers: For faster web testing
  • Retry Flaky Tests: Handle intermittent failures
  • Generate Reports: HTML, JUnit, Allure reports
  • Database Cleanup: Reset state between test runs
  • Configuration Management: Environment-specific configs

Securing Jenkins

# Jenkins Configuration: 1. Enable Security: - Manage Jenkins > Configure Global Security - Enable authentication - Set authorization strategy 2. Use Credentials: - Manage Jenkins > Manage Credentials - Store API keys, passwords securely - Reference in pipeline using ${CREDENTIAL} 3. Restrict Access: - Use role-based access control - Limit who can create/modify jobs - Restrict agent access 4. Audit Logging: - Enable audit logging - Monitor job executions - Track configuration changes # Example: Using credentials in pipeline pipeline { agent any environment { API_KEY = credentials('api-key-credential-id') DB_PASSWORD = credentials('db-password-credential-id') } stages { stage('Run Tests') { steps { sh ''' export API_KEY=${API_KEY} pytest tests/ -v ''' } } } }

Monitoring & Maintenance

# Regular Maintenance Tasks: 1. Monitor Disk Space: - Archive old builds - Clean up workspace regularly - Use build discarder 2. Update Plugins: - Check for plugin updates - Test updates in dev environment - Schedule updates during maintenance window 3. Backup Configuration: - Regularly backup Jenkins configuration - Store backups securely - Test restore procedure 4. Performance Tuning: - Monitor executor utilization - Adjust number of executors - Add agents for distributed testing 5. Logging & Alerts: - Monitor build failures - Set up email/Slack notifications - Analyze logs for issues

Troubleshooting Common Issues

# Common Jenkins Issues: 1. Build Timeout Problem: Builds hang indefinitely Solution: Add timeout(time: X, unit: 'HOURS') to pipeline 2. Out of Disk Space Problem: Jenkins fails to archive artifacts Solution: Enable build discarder, clean old builds 3. Authentication Failed Problem: Git/API authentication issues Solution: Check credentials, regenerate tokens 4. Flaky Tests Problem: Tests pass sometimes, fail other times Solution: Implement retry logic, fix test isolation 5. Slow Builds Problem: Pipeline takes too long Solution: Parallelize stages, optimize test suite 6. Agent Connection Issues Problem: Agents disconnect from master Solution: Check network, increase timeouts, restart agents
